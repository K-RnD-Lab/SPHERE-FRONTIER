Блок 07

🔐 Кібербезпека і криптографія

Канон PDF v25, стор. 56–58. Ціль: розвести hash, encryption, signature, authentication, authorization, TLS.

CIA ↔ hash ↔ encryption ↔ signature ↔ TLS ↔ access

Hash        → відбиток, односторонній, не decrypt
Encryption  → encrypt/decrypt з ключем (AES, Калина)
Signature   → авторство + цілісність
Authn       → хто ти
Authz       → що тобі можна
HTTPS       → HTTP поверх TLS

Українські алгоритми (пастка)

  • Купина — криптографічна hash-функція
  • Калина, Струмок, AES/Rijndael — шифри (не hash)

SIEM і SSL/TLS (ЄФВВ SETup)

Теми з бота SET, які доповнюють блок 07 для магістратури SET / ЄФВВ.

SIEM (Security Information and Event Management)

  • Збирає події з мережевих пристроїв, серверів, БД, датчиків безпеки, застосунків
  • Ключовий механізм: автентифікація джерел подій (не «просто шифрування»)
  • Атомарне спрацювання IDS: оповіщення при кожному збігу сигнатури

SSL/TLS

  • Механізми: шифрування, автентифікація, цілісність, цифровий підпис
  • На іспиті TLS часто «прив’язують» до транспортного рівня (захищені сесії веб/пошта)
  • HTTPS = HTTP + TLS (канал, не «безпека всього сайту»)

📊 Ризик-менеджмент (піраміда стратегій)

Блок питань SET bot (≈30–75): управління інформаційними ризиками.

Оцінка
  активи → загрози → вразливості → ризик
  Risk exposure = ймовірність × потенційні втрати

4 стратегії (зверху вниз — від «жити з ризиком» до «прибрати»):
  1. Прийняти (accept)   — усвідомлене рішення керівництва, прийнятний рівень
  2. Зменшити (mitigate) — технічні + організаційні + адміністративні контролі
  3. Передати (transfer)  — страхування, аутсорс (не зменшує P атаки!)
  4. Уникнути (avoid)    — відмова від активності / системи

Компенсація прийнятого ризику: резервні копії, плани реагування, страховки

Якісний vs кількісний аналіз

  • Якісний — експертні оцінки (суб’єктивність — головна проблема)
  • Кількісний — математика (Монте-Карло, historical data)

Пастки

⚠ Типові пастки (кібер + ризики)

  1. SIEM → шифрування ❌ → автентифікація джерел
  2. Страхування зменшує атаки ❌ → лише компенсує збитки
  3. Hash можна розшифрувати ❌ → односторонній відбиток
  4. Encoding (Base64) = encryption ❌
  5. Загрози лише «зовнішні / внутрішні» ❌ → ще природні
  6. Друкарські пастки: Bublic, primate — читай уважно
