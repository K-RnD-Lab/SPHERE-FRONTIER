🔬 Case: How to customize this environment

This page reads session data from the K-Mentorship-Hub Master Training module. Sessions are stored in localStorage on the client side and optionally synced to Google Sheets via Apps Script. If you want to replicate this setup for your own lab or classroom:

📦 Client-side storage

Sessions auto-save to localStorage key mt_sessions. No backend needed — works on any static host (Vercel, Netlify, GitHub Pages).

const sessions = JSON.parse(localStorage.getItem("mt_sessions") || "[]");

📊 Google Sheets sync

Set APPS_SCRIPT_URL in trainer.js to your deployed Apps Script web app URL. Sessions will POST there on completion.

let APPS_SCRIPT_URL = "https://script.google.com/.../exec";

🔑 Credentials

Change SHEET_ID in trainer.js to your Google Sheet ID. The sheet must be "Anyone with link can view" for reading.

const SHEET_ID = "your-sheet-id-here";

🧬 Cross-site display

This R&D Lab page reads from the same localStorage (same domain) or from Google Sheets. To share across domains, use the Sheets integration.

await fetch(SHEET_URL) // gviz API for reading
